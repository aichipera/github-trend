# [幽灵代码：一个被遗忘的Bug，如何差点颠覆谷歌的“反广告霸权”](20250713-google_web_reqblock_hack.mp3)

在一场关乎网络浏览自由和广告拦截未来的技术角力中，谷歌凭借其Chrome浏览器的绝对优势，强力推行了Manifest V3（简称MV3）扩展标准。这一举措的核心，是移除了名为`webRequestBlocking`的关键API，该API是uBlock Origin等主流广告拦截工具赖以生存的基石。此举被广泛解读为谷歌利用其平台权力，为自己庞大的广告业务“扫清障碍”。然而，正如一篇引人入胜的[技术博文](https://0x44.xyz/blog/web-request-blocking/)所揭示，一个早已废弃的代码片段，竟成了这堵看似坚不可摧的MV3高墙上的一个“幽灵后门”。

## 一、最有趣的Bug：用“前朝”的剑，斩“本朝”的官

故事的主角是一位安全研究员，他在2023年发现并报告了这个漏洞。其核心在于，尽管MV3严格禁止了`webRequestBlocking`权限，但Chrome中仍残留着为处理已于2020年被弃用的“平台应用（Platform Apps）”而存在的旧有JavaScript绑定代码。

这些旧代码中存在一个特殊参数`opt_webViewInstanceId`，它允许平台应用为其嵌入的WebView（网页视图）开启请求拦截功能，以便于管理。关键的疏漏在于，当一个扩展创建自己的“伪造”`webRequest`事件并带上这个ID时，浏览器后台的权限检查系统竟然“开绿灯”了——它跳过了对`webRequestBlocking`权限的验证。

换句话说，任何MV3扩展都可以通过下面这样的几行代码，“假扮”成一个古老的平台应用，从而重新获得强大的动态请求拦截能力，让谷歌的MV3限制瞬间失效：

```javascript
let WebRequestEvent = chrome.webRequest.onBeforeRequest.constructor
// opt_webViewInstanceId是第5个参数
let fakeEvent = new WebRequestEvent("webRequest.onBeforeRequest", 0, 0, 0, 1337)
fakeEvent.addListener(() => {
    return { cancel: true }
}, { urls: ['*://*.example.com/*'] }, ['blocking'])
```

一个本应被历史尘封的特性，却成了绕过当前严格限制的“万能钥匙”。这个漏洞的存在，无疑是对谷歌宏大更新计划的巨大讽刺。作者也坦言，他不知道如何制作广告拦截器，于是将问题报告给了谷歌，最终该漏洞在Chrome 118版本中被修复，而他获得的赏金是——0美元，因为谷歌认为这不构成安全问题。

## 二、Chrome的围城：我们为何被困在“最好”的浏览器里？

这个漏洞的故事迅速在[Hacker News上引发了激烈的讨论](https://news.ycombinator.com/item?id=44544266)，焦点很快从技术细节转向了更深层次的平台垄断问题。许多评论者一针见血地指出，Chrome已经成为新的“IE浏览器”——不是因为它技术落后，恰恰相反，是因为它过于强大和普及，以至于正在单方面定义网络标准。

一位开发者无奈地表示：“许多开发者将‘能在Chrome中运行’等同于‘能用’，即使这是利用了Chrome的特性或怪癖。”这种“Chrome即标准”的心态，导致大量网站和应用在Firefox或其他非Chromium内核的浏览器上体验不佳甚至无法使用。有用户反馈，自己国家的银行网站甚至都无法在Firefox上正常完成交易，迫使他不得不“投降”，全面转向Chrome。

这形成了一个难以打破的恶性循环：用户因为兼容性问题选择Chrome -> 开发者因为用户份额集中于Chrome而优先为其开发 -> 进一步加剧了其他浏览器的兼容性问题 -> 更多用户选择Chrome。

## 三、无力的反抗？“换用Firefox”为何知易行难

面对谷歌的“霸权”，最直接的反抗似乎就是“用脚投票”，转向Firefox等替代品。许多用户分享了他们切换到Firefox后的良好体验，尤其是其强大的多账户容器（Multi-Account Containers）功能和对广告拦截的坚定支持。

然而，这条路也并非坦途。一些人指出，即便Firefox很好，但它也无法摆脱谷歌的阴影。Mozilla基金会相当一部分收入来源于谷歌，以换取后者成为其默认搜索引擎。有人犀利地评论道：“谷歌每年花几亿美元资助Mozilla，就像是买了一块‘遮羞布’，好在标准委员会面前假装自己不是垄断者。”

更现实的困境在于，即便个人用户下定决心切换，工作环境和特定网站（如银行、某些政府服务）的强制要求，往往让他们不得不保留一个Chromium内核的浏览器。这种无力感，使得个体的反抗显得杯水车薪。

## 四、拆分巨兽：唯一的终局？

当一个漏洞的发现，最终指向的是一场关于平台控制权的哲学辩论时，解决方案或许已超越了技术本身。Hacker News上的一位用户详尽列举了谷歌利用其生态系统进行全方位控制的“罪状”，从搜索、广告到操作系统、应用商店，乃至支付和账户体系，形成了一个密不透风的商业闭环。他总结道：“许多大型科技公司都应面临同样的审判，但没有哪家像谷歌一样厚颜无耻和具有吸血性。”

这个发现于无意间的“幽灵代码”，就像一道闪电，短暂地照亮了这座由代码和商业利益构建的庞大帝国的地基。它提醒我们，用户的权利和网络的开放性，正被一点点蚕食。漏洞可以被修复，但由垄断带来的系统性问题，却需要更深刻的反思和更强力的行动。
